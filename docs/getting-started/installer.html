How Ifá-Lang installation works across platforms.

Crates

Crate Purpose

Core Modules

Source: crates/ifa-installer-core/src/

Module Purpose

uninstall.rs Clean removal
extraction.rs Archive extraction
net.rs Download from release servers
check.rs Verify installation
config.rs Installation configuration
profiles.rs Installation profiles (minimal, full)
args.rs CLI argument parsing
windows.rs Windows-specific (PATH, registry)
unix.rs Unix-specific (symlinks, permissions)

Installation Profiles

Profile Includes
minimal ifa CLI only
standard CLI + standard library
full CLI + std + all stacks + embedded
development Full + Babalawo + documentation

Installation Flow

1. Check prerequisites (Rust, OS version)
2. Download release archive from GitHub
3. Verify checksum
4. Extract to target directory
5. Platform setup:
   - Windows: Add to PATH, create Start Menu entry
   - Unix: Create symlinks, set shell rc
6. Verify installation
7. Run post-install hooks

CLI Usage

# Install latest stable
ifa-installer install

# Install specific version
ifa-installer install --version 1.2.2

# Install with profile
ifa-installer install --profile development

# Check installation
ifa-installer check

# Uninstall
ifa-installer uninstall

Rollback

The installer uses RAII for automatic rollback on failure:

// If any step fails, previous steps are undone:
// - Extracted files are deleted
// - PATH changes are reverted
// - Registry entries are removed
