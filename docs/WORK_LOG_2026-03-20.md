## 2) Implementation Changes (Rust)

### 2.1 Bytecode compiler: jump encoding + logical short-circuit

File: `crates/ifa-core/src/compiler.rs`

Changes made:
- **Jump operand encoding fixed to 32-bit absolute offsets** (little-endian) to match the VM (`Jump`, `JumpIfTrue`, `JumpIfFalse`) and `ifa-bytecode`’s operand sizing.
- Updated loop back-jumps (`while`, `for`) to emit absolute targets instead of 16-bit relative negative offsets.
- Implemented **short-circuit compilation** for logical `&&` / `||` with operand-return semantics by using:
  - `Dup` + `JumpIfFalse` / `JumpIfTrue` + `Pop` + RHS compilation + patching.

Rationale:
- The VM reads `u32` jump targets and treats them as absolute instruction pointers; the prior 16-bit relative patching was incompatible and a major control-flow blocker.
- Spec requires short-circuiting and returning the operand value, not a Bool.

### 2.2 VM comparisons: enforce spec-mandated TypeError on incompatible types

File: `crates/ifa-core/src/vm.rs`

Changes made:
- Updated `Lt` / `Le` / `Gt` / `Ge` to raise `IfaError::TypeError` when operands are not among the defined comparable types (Int/Float/String), instead of returning `false`.

Rationale:
- Spec §6.6: “Comparing incompatible types MUST produce a TypeError.”

## 4) Known Open Issues / Next Steps

High-impact items that remain after this pass:
- **Why `runb` hits an import path** (`Imports not implemented in registry`) for `examples/vm_smoke.ifab` even though the source contains no import statements.
  - Suspected causes include instruction stream misalignment, an unexpected emitted `Import`, or a wrong jump target landing in the middle of operands.
  - Next step: finish the `.ifab` disassembler and confirm the executed opcode sequence.
- **`Import` opcode format mismatch** between `ifa-bytecode` (marked variable-length) and `ifa-core` compiler/VM (currently encode/decode as fixed `u16` string index).
  - Requires either: updating `ifa-bytecode`’s `operand_bytes()` for `Import`, or implementing a variable-length operand layout in `ifa-core`.
- **Std registry import support**: `StdRegistry` does not override `OduRegistry::import` (default is “not implemented”), so any legitimate `Import` opcode will currently fail in the VM path.
