#[derive(Ebo)] - Resource Sacrifice

Automatically implements Drop trait for RAII resource cleanup. The macro embodies the Yoruba concept of  (sacrifice/offering) where resources must be properly returned to the system.

Basic Usage

Ebo Derive 
#[derive(Ebo)]
struct MyFile {
    handle: std::fs::File,
}

// Drop is auto-implemented:
impl Drop for MyFile {
    fn drop(&mut self) {
        println!("[Ebo] Sacrificed: MyFile");
    }
}

Custom Cleanup Method

Custom Ebo 
#[derive(Ebo)]
#[ebo(cleanup = "close")]
struct DatabaseConnection {
    pool: sql::Pool,
}

// Generates:
impl Drop for DatabaseConnection {
    fn drop(&mut self) {
        println!("[Ebo] Sacrificing DatabaseConnection");
        self.close();  // Calls custom cleanup method
    }
}

Attribute Parameters

  • cleanup = "method" - Specify custom cleanup method name
  • Default: Prints sacrifice message without cleanup

#[iwa_pele] - Gentle Balance

Compile-time balance checking for paired operations. Ìwà Pélé (gentle character) ensures that resources opened in a scope are properly closed in the same scope, preventing resource leaks.

Checked Operation Pairs

  • so / pa - socket open/close
  • si / ti - file open/close
  • mu / fi - acquire/release
  • bere / da - start/stop

Usage Example

Balance Checking 
#[iwa_pele]
fn network_task() {
    let conn = Otura.so("example.com", 80);  // open
    // ... work with connection
    conn.pa();  // close - REQUIRED or compile error
}

// This would cause compile error:
#[iwa_pele]
fn bad_function() {
    let file = Odi.si("data.txt");  // open file
    // Missing: file.ti() - compile error!
    // Ìwà Pélé violation: Block leaks resource
}

Compile-Time Enforcement

The macro performs lexical CFG (Control Flow Graph) analysis at compile time:

  • Zero Runtime Cost: No runtime overhead
  • Lexical Scope: Checks within each block
  • Clear Errors: Detailed violation messages
  • Early Detection: Catches leaks before execution

ajose! - Reactive Binding

Creates reactive bindings between values. Àjõsé (gathering/assembly) represents the reactive system where changes propagate through relationships automatically.

Basic Reactive Binding

Standard Binding 
let counter = RefCell::new(0);
let label = RefCell::new("0".to_string());

// Create reactive binding
ajose!(counter.borrow() => label.borrow_mut());

// When counter changes, label updates automatically
*counter.borrow_mut() = 42;  // label becomes "42"

Freeze Binding (Cross-thread)

Thread-Safe Binding 
let shared_data = Arc::new(Mutex::new(Vec::new()));
let local_counter = RefCell::new(0);

// Auto-freeze for cross-thread safety
ajose!(local_counter.borrow() => #freeze shared_data.lock().unwrap());

// Creates frozen value safe for sharing across threads

Generated Code

The macro generates subscription logic and initial synchronization:

  • Subscription Creation: Callback for value changes
  • Initial Sync: Sets target to source value
  • Freeze Logic: Optional thread-safe freezing
  • Debug Output: Binding creation logging

ebo_block! - Scoped Resource Management

Creates a scoped RAII block with guaranteed cleanup. The block embodies the concept of ritual boundaries where resources are managed within a sacred space.

Usage

Ebo Block 
ebo_block! {
    let file = std::fs::File::open("data.txt")?;
    let conn = Database::connect("localhost")?;
    
    // Work with resources
    process_data(&file, &conn)?;
    
    // All resources auto-cleanup when block exits
}

// Output:
// --- Ebo Block Started ---
// [Work happens here]
// --- Ebo Block Complete ---

Generated Structure

The macro creates a guard struct with Drop implementation:

  • Guard Creation: _EboGuard instance
  • Start Logging: Block initiation message
  • Drop Implementation: Completion logging
  • Scope Safety: Guaranteed cleanup

#[derive(Observable)] - Reactive Data

Makes a struct's fields observable for reactive updates. Generates watcher methods for each field, enabling reactive programming patterns.

Basic Usage

Observable Struct 
#[derive(Observable)]
struct Counter {
    value: i32,
    name: String,
}

// Generated methods:
impl Counter {
    pub fn watch_value(&self, callback: F) {
        println!("[Observable] Watching: Counter.value");
        callback(&self.value);
    }
    
    pub fn watch_name(&self, callback: F) {
        println!("[Observable] Watching: Counter.name");
        callback(&self.name);
    }
}

Reactive Pattern

Reactive Usage 
let counter = Counter { value: 0, name: "counter".to_string() };

// Watch for changes
counter.watch_value(|val| {
    println!("Value changed to: {}", val);
});

counter.watch_name(|name| {
    println!("Name changed to: {}", name);
});

Generated Features

  • Field Watchers: watch_{field_name} methods
  • Type Safety: Generics for callback types
  • Logging: Debug output for watcher creation
  • Struct Only: Works only on struct types

Integration with Runtime Specification

Cultural Safety

These macros embody Yoruba cultural principles:

  •  (Sacrifice): Resources must be properly returned
  • Ìwà Pélé (Gentle Character): Balance and harmony in operations
  • Àjõsé (Gathering): Community of reactive relationships
  • Sacred Boundaries: Scoped resource management

Protocol-Oriented Design

Macros support the protocol-oriented architecture:

  • No Classes: Derive macros work with struct protocols
  • Capability-Based: Resources implement cleanup protocols
  • Structural Typing: Balance checking based on method names
  • Reactive Protocols: Observable and binding capabilities

Three-Tier Runtime Support

  • AST Interpreter: Macro expansion at compile time
  • Bytecode VM: Zero-cost abstractions
  • Transpiler: Generated code compatible with all targets

Best Practices

Resource Management

  • Use #[derive(Ebo)] for all resource-holding structs
  • Apply #[iwa_pele] to functions with paired operations
  • Wrap critical sections with ebo_block!
  • Specify custom cleanup methods when needed

Reactive Programming

  • Make data structures #[derive(Observable)] for UI updates
  • Use ajose! for cross-component data binding
  • Apply #freeze modifier for cross-thread bindings
  • Combine with async patterns for reactive I/O

Performance Considerations

  • Zero Cost: #[iwa_pele] has no runtime overhead
  • Compile Time: All checks happen during compilation
  • Minimal Runtime: Other macros have minimal overhead
  • Optimization Friendly: Generated code optimizes well

Example: Complete Application

Integrated Macro Usage 
// Observable data structure
#[derive(Observable)]
struct AppState {
    counter: i32,
    status: String,
}

// Resource with automatic cleanup
#[derive(Ebo)]
#[ebo(cleanup = "disconnect")]
struct DatabaseConnection {
    pool: sql::Pool,
}

impl DatabaseConnection {
    fn disconnect(&mut self) {
        // Custom cleanup logic
        self.pool.close();
    }
}

#[iwa_pele]
fn main() {
    ebo_block! {
        let conn = DatabaseConnection::new();
        let state = AppState { counter: 0, status: "ready".to_string() };
        
        // Reactive binding
        ajose!(state.counter => conn.query_count);
        
        // Watch for changes
        state.watch_status(|status| {
            println!("Status: {}", status);
        });
        
        // Update state (triggers reactive updates)
        state.counter = 42;
        state.status = "processing".to_string();
        
        // conn automatically disconnected on block exit
    }
}
