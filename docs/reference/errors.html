Error Reference

IfáError System with Cultural Wisdom

Overview

Ifá-Lang uses a unified error system through IfaError with Yoruba proverbs and structured error codes. All runtime components use the same error type, ensuring consistency across the entire ecosystem.

Error Categories

Math Errors (Obara / Oturupon) - ErrorCode 0x03XX

DivisionByZero

Proverb: "Obara rejects: {context}"

Code: 0x0301

// Causes error
ayanmo result = 10 / 0;

// Error message
// DivisionByZero - Obara rejects: Cannot divide by zero

Overflow

Proverb: "Numeric overflow: {context}"

Code: 0x0302

// Causes error
ayanmo big = i64::MAX + 1;

// Error message  
// Overflow - Numeric overflow: Addition exceeds maximum value

Underflow

Proverb: "Numeric underflow: {context}"

Code: 0x0303

// Causes error
ayanmo small = i64::MIN - 1;

// Error message
// Underflow - Numeric underflow: Subtraction below minimum value

Type Errors - ErrorCode 0x02XX

ArityMismatch

Proverb: "Wrong number of arguments"

Code: 0x0201

// Function expects 2 args, gets 1
ese add(a, b) { pada a + b; }
ayanmo result = add(5);

// Error message
// ArityMismatch: expected 2 arguments, got 1

ArgumentError

Proverb: "Invalid argument: {context}"

Code: 0x0202

// Invalid argument value
ayanmo result = Obara.akako(-5);

// Error message
// ArgumentError: Cannot calculate square root of negative number

TypeError

Proverb: "Expected {expected}, got {got}"

Code: 0x0203

// Type mismatch in operation
ayanmo result = "hello" * 5;

// Error message
// TypeError: expected Number, got String

ConversionError

Proverb: "Cannot convert {from} to {to}"

Code: 0x0204

// Invalid conversion
ayanmo num = Ika.nomba("not_a_number");

// Error message
// ConversionError: Cannot convert String to Int

Index/Key Errors (Ogunda / Ika) - ErrorCode 0x01XX

IndexOutOfBounds

Proverb: "Index {index} out of bounds (length: {length})"

Code: 0x0101

Proverb: "Key '{key}' not found in map"

Code: 0x0102

// Error message
// KeyNotFound: Key 'age' not found in map

Error Handling Patterns

Try-Catch-Finally

Error Handling 
// Yoruba style
gbiyanju {
    ayanmo result = risky_operation();
} gba (error) {
    Irosu.fo("Error occurred: " + error.message);
    // Handle specific error types
    if error.code == 0x0301 {
        Irosu.fo("Division by zero detected");
    }
} nipari {
    cleanup_resources();
}

// English style
try {
    ayanmo result = risky_operation();
} catch (error) {
    Irosu.fo("Error occurred: " + error.message);
} finally {
    cleanup_resources();
}

Result Types

Result Pattern 
// Function that returns Result
ese safe_divide(a, b) -> IfaResult {
    if b == 0 {
        ku(IfaError::DivisionByZero("Cannot divide by zero".to_string()));
    }
    pada a / b;
}

// Using Result
match safe_divide(10, 2) {
    Ok(result) => Irosu.fo("Result: " + result),
    Ku(error) => Irosu.fo("Error: " + error.message),
}

Error Prevention

Ìwà Pélé Balance Checking

Compile-time resource balance checking prevents resource leaks:

Balance Checking 
#[iwa_pele]
fn network_task() {
    let conn = Otura.so("example.com", 80);  // open
    // ... work with connection
    conn.pa();  // close - REQUIRED or compile error
}

// This would cause compile error:
#[iwa_pele]
fn bad_function() {
    let file = Odi.si("data.txt");  // open file
    // Missing: file.ti() - compile error!
    // Ìwà Pélé violation: Block leaks resource
}

Type Safe Operations

Use type checking to prevent runtime errors:

Type Safety 
ese safe_operation(value) {
    // Check types before operations
    if Oka.is_num(value) {
        // Safe numeric operation
        pada value * 2;
    } else if Oka.is_str(value) {
        // Safe string operation
        pada Ika.length(value);
    } else {
        // Clear error message
        ku(IfaError::TypeError(
            "Expected Number or String, got ".to_string() + Oka.type_name(value)
        ));
    }
}

Error Codes Reference

Code Ranges

  • 0x01XX: Index/Key errors (Ogunda/Ika)
  • 0x02XX: Type errors (General)
  • 0x03XX: Math errors (Obara/Oturupon)
  • 0x04XX: Runtime errors (General)
  • 0x05XX: File/Storage errors (Odi)
  • 0x06XX: Network errors (Otura)
  • 0x07XX: Crypto errors (Irete)
  • 0x08XX: Graphics errors (Ose)
  • 0x09XX: Permission errors (Ofun)

Best Practices

Error Handling Guidelines

  • Be Specific: Use appropriate error types
  • Provide Context: Include relevant information in error messages
  • Handle Early: Handle errors as close to source as possible
  • Log Errors: Use Irosu for error logging
  • Recover Gracefully: Provide fallback behavior when possible

Error Message Style

  • Yoruba Proverbs: Include cultural wisdom when appropriate
  • Clear Context: Explain what went wrong and why
  • Actionable: Suggest how to fix the error
  • Consistent: Follow established patterns

Performance Considerations

  • Zero Cost: Use compile-time checks when possible
  • Early Returns: Fail fast to avoid wasted work
  • Error Pooling: Reuse error objects in hot paths
  • Structured Errors: Use error codes for efficient handling
