Type System Reference

Dynamic Types with Protocol-Oriented Design

Overview

Ifá-Lang uses a dynamic type system with protocol-oriented design. All values are instances of IfaValue, which provides a unified container for different data types while maintaining type safety through structural subtyping.

Core Types

Primitive Types

Int

64-bit signed integer

ayanmo x = 42;
ayanmo negative = -1000;
  • Range: -9,223,372,036,854,775,808 to 9,223,372,036,854,775,807
  • Size: 8 bytes
  • Methods: All Obara math operations

Float

64-bit floating point number

ayanmo pi = 3.14159;
ayanmo scientific = 1.23e-4;
  • IEEE 754 double precision
  • Size: 8 bytes
  • Methods: All Obara math operations

Bool

Boolean value

ayanmo truth = otito;
ayanmo lies = eke;
  • Values: otito (true), eke (false)
  • Size: 1 byte
  • Methods: Logical operations

Str

Unicode string

ayanmo name = "Ifá-Lang";
ayanmo yoruba = "Kaabo àwujú!";
  • UTF-8 encoded
  • Immutable by default
  • Methods: Ika string operations

Collection Types

List

Ordered collection of values

ayanmo numbers = [1, 2, 3, 4, 5];
ayanmo mixed = [1, "hello", otito, 3.14];
  • Dynamic typing (mixed types allowed)
  • Zero-based indexing
  • Methods: Ogunda array operations

Map

Key-value dictionary

ayanmo person = {
    "name": "Alice",
    "age": 30,
    "active": otito
};
  • String keys only
  • Any value types
  • Methods: Owonrin map operations

Object

Thread-local mutable object ("The Hut")

ayanmo obj = Object();
obj.field = "value";
ayanmo value = obj.field;
  • Thread-local (no atomic overhead)
  • Mutable fields
  • Methods: Standard object operations

Function Types

Function

Callable function reference

ese add(a, b) {
    pada a + b;
}
ayanmo func = add;
ayanmo result = func(5, 3);
  • First-class functions
  • Closure support
  • Higher-order functions

Resource

Opaque resource handle

ayanmo file = Odi.si("data.txt");
// file is a ResourceToken
  • Opaque handles
  • Automatic cleanup
  • Domain-specific operations

Null

Null/empty value

ayanmo empty = ofo;
ayanmo optional = get_maybe_null();
  • Represents absence of value
  • Used in optional contexts
  • Methods: Null-safe operations

Type Annotations

Type Hints

Optional type hints can be added for clarity:

Type Annotations 
// With type hints
ayanmo name: Str = "Alice";
ayanmo age: Int = 30;
ayanmo active: Bool = otito;

// Function parameters
ese greet(name: Str) -> Str {
    pada "Kaabo, " + name;
}

// Collection types
ayanmo numbers: List[Int] = [1, 2, 3];
ayanmo person: Map[Str, Any] = {"name": "Bob"};

Available Type Names

  • Primitives: Int, Float, Str, Bool
  • Collections: List, Map, Object
  • Numeric: i8, i16, i32, i64, u8, u16, u32, u64
  • Float: f32, f64
  • Universal: Any, void

Type Conversion

Implicit Conversion

Some conversions happen automatically:

Automatic Conversions 
// Numeric promotion
ayanmo result = 5 + 3.14;  // Int -> Float

// String concatenation
ayanmo message = "Value: " + 42;  // Int -> Str

// Boolean context
ayanmo truthy = 42;        // Non-zero -> otito
ayanmo falsy = 0;          // Zero -> eke

Explicit Conversion

Use domain methods for explicit conversion:

Explicit Conversions 
// String to number
ayanmo num = Ika.nomba("123");

// Number to string  
ayanmo str = Ika.kika(42);

// Type checking
ayanmo is_int = Oka.is_int(42);
ayanmo is_str = Oka.is_str("hello");

Protocol-Oriented Types

Type Protocols

Types implement protocols rather than inheritance:

Protocol Implementation 
// Any type that can be added implements "Addable"
// Any type that can be printed implements "Printable"
// Any type that can be indexed implements "Indexable"

// Usage is based on capability, not type
ese process(value) {
    if value.has_capability("Printable") {
        Irosu.fo(value);
    }
}

Structural Subtyping

Types are compatible based on structure, not declaration:

Structural Compatibility 
// These are structurally the same
ayanmo person1 = {"name": "Alice", "age": 30};
ayanmo person2 = {"name": "Bob", "age": 25};

// Can be used interchangeably if they have same fields
ese display(info) {
    Irosu.fo(info.name + " (" + info.age + ")");
}

Special Types

VM-Specific Types

Available when VM feature is enabled:

  • AstFn: AST function representation
  • BytecodeFn: Bytecode function representation
  • Ptr: Raw pointer (unsafe)
  • Ref: Immutable reference

Return Type

Special type for function return values:

Return Values 
ese calculate() {
    if some_condition {
        pada 42;  // Return(Int)
    }
    // Implicit return of ofo
}

Type Performance

Memory Layout

  • Tagged Union: Efficient discriminant-based dispatch
  • Arc<str>: String interning for shared strings
  • "The Hut": Thread-local objects with zero atomic overhead
  • Resource Tokens: Opaque handles for safe resource management

Optimization Tips

  • String Interning: Reuse strings when possible
  • Thread-Local Objects: Use Object for thread-local data
  • Type Hints: Help compiler optimize
  • Avoid Mixed Lists: Homogeneous lists are faster

Type Safety

Runtime Type Checking

Dynamic typing with runtime checks:

Type Safety 
// Type checking
ayanmo value = get_value();
if Oka.is_int(value) {
    // Safe to use as Int
    ayanmo doubled = value * 2;
}

// Safe operations with type guards
ese safe_add(a, b) {
    if Oka.is_num(a) and Oka.is_num(b) {
        pada a + b;
    } else {
        Okanran.ku("Both arguments must be numbers");
    }
}

Error Handling

Type errors are caught at runtime:

  • Type Mismatch: Wrong type for operation
  • Conversion Error: Invalid type conversion
  • Index Bounds: Collection access errors
  • Arity Mismatch: Wrong number of arguments
