Migrating from Python

Python Developer's Guide to Ifá-Lang

Overview

Ifá-Lang shares Python's dynamic typing philosophy but introduces cultural concepts, protocol-oriented design, and the 16 Odù domain system. This guide helps Python developers transition effectively.

Key Differences

Python

Object-oriented with classes and inheritance

Ifá-Lang

Protocol-oriented with Odù domains

Python

Dynamic typing with duck typing

Ifá-Lang

Dynamic typing with structural subtyping

Python

Standard library with modules and packages

Ifá-Lang

16 Odù domains (built-in, no imports)

Python

try/except/finally error handling

Ifá-Lang

gbiyanju/gba/nipari with cultural proverbs

Basic Syntax Comparison

Variables and Functions

Python 
# Variables
x = 5
name = "Alice"
PI = 3.14159  # Constants by convention

# Functions
def add(a, b):
    return a + b

# Lambda functions
multiply = lambda a, b: a * b

# Type hints
def greet(name: str) -> str:
    return f"Hello, {name}"
Ifá-Lang 
// Variables
ayanmo x = 5;
ayanmo name = "Alice";
const PI = 3.14159;  // Real constants

// Functions
ese add(a, b) {
    pada a + b;
}

// No lambda equivalent - use regular functions
ese multiply(a, b) {
    pada a * b;
}

// Type hints
ese greet(name: Str) -> Str {
    pada "Hello, " + name;
}

Control Flow

Python 
# If statement
if x > 0:
    print("Positive")
elif x < 0:
    print("Negative")
else:
    print("Zero")

# For loop
for i in range(10):
    print(i)

# While loop
while x < 10:
    x += 1

# List comprehension
squares = [x**2 for x in range(10)]
Ifá-Lang 
// If statement
bi x > 0 {
    Irosu.fo("Positive");
} bi kosi x < 0 {
    Irosu.fo("Negative");
} bi kosi {
    Irosu.fo("Zero");
}

// For loop
fun i ninu 0..10 {
    Irosu.fo(i);
}

// While loop
nigita x < 10 {
    x = x + 1;
}

// List operations (Ogunda domain)
ayanmo squares = Ogunda.map(0..10, |x| x * x);

Error Handling

gbiyanju {

} nipari {
    cleanup();
}

Advanced Concepts

Classes vs Odù Domains

Python 
class Person:
    def __init__(self, name, age):
        self.name = name
        self.age = age
    
    def greet(self):
        return f"Hello, I'm {self.name}"
    
    def __str__(self):
        return f"Person({self.name}, {self.age})"

class Student(Person):
    def __init__(self, name, age, grade):
        super().__init__(name, age)
        self.grade = grade

alice = Student("Alice", 30, "A")
print(alice.greet())
Ifá-Lang 
odu Person {
    ayanmo name: Str;
    ayanmo age: Int;
    
    ese greet() -> Str {
        pada "Hello, I'm " + this.name;
    }
    
    ese to_string() -> Str {
        pada "Person(" + this.name + ", " + this.age + ")";
    }
}

// No inheritance - use composition
odu Student {
    ayanmo person: Person;
    ayanmo grade: Str;
    
    ese greet() -> Str {
        pada this.person.greet() + " (Grade: " + this.grade + ")";
    }
}

ayanmo alice = Student();
alice.person = Person();
alice.person.name = "Alice";
alice.person.age = 30;
alice.grade = "A";
Irosu.fo(alice.greet());

Decorators vs Macros

Python 
# Decorators
def timer(func):
    def wrapper(*args, **kwargs):
        start = time.time()
        result = func(*args, **kwargs)
        end = time.time()
        print(f"Function took {end - start} seconds")
        return result
    return wrapper

@timer
def slow_function():
    time.sleep(1)

# Property decorators
class Person:
    @property
    def age_in_months(self):
        return self.age * 12
Ifá-Lang 
// Macro-based approach
#[derive(Ebo)]
#[ebo(cleanup = "close")]
struct FileResource {
    handle: FileHandle,
}

// Attribute macros
#[iwa_pele]
fn balanced_function() {
    let file = Odi.si("data.txt");
    // File automatically closed
}

// No direct decorator equivalent
// Use function composition or domain methods

Context Managers vs Resource Management

Python 
class FileContext:
    def __enter__(self):
        self.file = open("data.txt", "r")
        return self.file
    
    def __exit__(self, exc_type, exc_val, exc_tb):
        self.file.close()

# Usage
with FileContext() as f:
    content = f.read()
    # File automatically closed
Ifá-Lang 
// Ebo block for resource management
ebo_block! {
    let file = Odi.si("data.txt");
    let content = file.read();
    // File automatically closed when block exits
}

// Or individual resource management
ebo file = Odi.si("data.txt");
let content = file.read();
// File automatically closed when scope ends

Unique Ifá-Lang Concepts

Resource Management ( )

Ifá-Lang has automatic resource management that's more explicit than Python's context managers:

Python vs Ifá-Lang 
# Python - manual or context manager
with open("data.txt") as f:
    content = f.read()
# File automatically closed

# Ifá-Lang - automatic with 
ebo file = Odi.si("data.txt");
let content = file.read();
// File automatically closed when scope ends

Balance Checking (Ìwà Pélé)

Compile-time checking for resource balance that doesn't exist in Python:

Balance Checking 
#[iwa_pele]
fn network_task() {
    let conn = Otura.so("example.com", 80);  // open
    // ... work with connection
    conn.pa();  // close - REQUIRED or compile error
}

// Missing conn.pa() would cause compile error!
// Python would have no such guarantee

Reactive Programming (Àjõsé)

Built-in reactive system that requires external libraries in Python:

Reactive Programming 
# Python - requires external libraries
import rx

# Ifá-Lang - built-in reactive system
ajose!(counter.value => label.text);

# When counter changes, label updates automatically
counter.value = 42;  // label.text becomes "42"

# No external libraries needed!

Protocol-Oriented Design

No classes or inheritance - uses protocols and structural typing:

Protocol Design 
// No inheritance - use composition
odu Database {
    ese query(sql: Str) -> List[Map] { ... }
    ese close() { ... }
}

// Any struct with query/close methods can be used as Database
// No need for explicit inheritance

Migration Strategy

Step 1: Learn the Syntax

  • Start with basic syntax (ayanmo, ese, bi, nigita)
  • Practice with Irosu domain for I/O operations
  • Get comfortable with semicolons and braces
  • Learn the bilingual keyword system

Step 2: Understand Domains

  • Learn the 16 Odù domains and their purposes
  • Replace Python standard library with domain methods
  • Practice domain-specific operations
  • Understand that domains are built-in (no imports)

Step 3: Adapt Python Patterns

  • Convert classes to Odù definitions
  • Replace decorators with macros where applicable
  • Adapt context managers to  blocks
  • Learn protocol-oriented design patterns

Step 4: Embrace New Concepts

  • Learn resource management ( )
  • Practice reactive programming (Àjõsé)
  • Understand balance checking (Ìwà Pélé)
  • Explore cultural philosophy integration

Common Pitfalls

Importing Domains

Wrong 
# WRONG - Don't import domains like Python modules
import std.irosu
Irosu.fo("Hello")
Correct 
// CORRECT - Use domains directly
Irosu.fo("Hello")

Class Inheritance

Python Style 
# Python inheritance - WRONG in Ifá-Lang
class Student(Person):
    def __init__(self, name, age, grade):
        super().__init__(name, age)
        self.grade = grade
Ifá-Lang Style 
// Ifá-Lang composition
odu Student {
    ayanmo person: Person;
    ayanmo grade: Str;
}
