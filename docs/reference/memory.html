The Opon (Calabash)

In Ifá tradition, the calabash (?p?n) is a sacred container. In Ifá-Lang, the Opon is the central memory container for the Virtual Machine.

Size Presets

Opon supports bilingual size names (Yoruba/English):

Size Yoruba English Slots Memory Use Case
Kekere kekere small, tiny, micro 256 ~4KB Embedded/IoT
Arinrin arinrin default, medium, standard 4,096 ~64KB Default
Nla nla large, big, mega 65,536 ~1MB Large workloads
Ailopin ailopin unlimited, dynamic 8 Dynamic No limits

Setting Memory Size

# CLI flag
ifa run --memory=nla myapp.ifa
ifa run --memory=large myapp.ifa  # English alias

# Embedded
ifa run --memory=kekere embedded.ifa

IfaValue Type

All values in Ifá-Lang are stored as IfaValue:

// From crates/ifa-core/src/value.rs
pub enum IfaValue {
    Nil,                          // No value
    Bool(bool),                   // otito/iro (true/false)
    Number(f64),                  // All numbers are 64-bit floats
    String(Rc),           // Reference-counted strings
    List(Rc>>),   // Mutable lists
    Map(Rc>>),     // Mutable maps
    Function(Rc),       // User-defined functions
    NativeFunction(Rc), // Built-in functions
}

// Error types
OponErrorKind::MemoryLimitExceeded  // Allocation beyond max
OponErrorKind::InvalidAddress       // Bad address access

Flight Recorder

The Opon includes a flight recorder for debugging. It stores the last 128 memory events:

// Event types recorded:
- Allocate(address, type)
- Set(address, old_type, new_type)
- Get(address)
- Free(address)

// Dump after crash
opon.dump_history();  // Writes to stderr

Garbage Collection

Ifá-Lang uses reference counting (Rc) for automatic memory management. When an IfaValue's reference count drops to zero, it is automatically freed.

For explicit resource cleanup (files, connections), use ?b? (RAII):

?b? file = Odi.open("data.txt");
// file is automatically closed when scope ends
