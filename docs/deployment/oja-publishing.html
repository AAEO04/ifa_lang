Publishing to Oja

Package Publishing Guide

Overview

Publishing packages to Oja (the Ifá-Lang marketplace) allows you to share your libraries, frameworks, and tools with the community. This guide covers the complete publishing process from package creation to release.

Main Library File

src/lib.ifa 
// Package metadata
/// Awesome utility functions for Ifá-Lang
/// 
/// This package provides common utilities for string manipulation,
/// data validation, and formatting.
/// 
/// # Examples
/// 
/// ```
/// ayanmo result = ifa_awesome_utils::format_name("alice");
/// Irosu.fo(result); // "Alice"
/// ```
/// 
/// # Features
/// 
/// - `std`: Standard library support (default)
/// - `serde`: Serialization support
/// - `async`: Async function support

// Public API
pub ese format_name(name: Str) -> Str {
    /// Format a name to title case
    /// 
    /// # Arguments
    /// 
    /// * `name` - The name to format
    /// 
    /// # Returns
    /// 
    /// Title-cased name
    Ika.tikun(Ika.first_char(name)) + Ika.substring(name, 1);
}

pub ese validate_email(email: Str) -> Bool {
    /// Validate email address format
    /// 
    /// # Arguments
    /// 
    /// * `email` - Email address to validate
    /// 
    /// # Returns
    /// 
    /// `otito` if valid, `eke` otherwise
    Ika.contains(email, "@") and Ika.contains(email, ".");
}

pub ese format_currency(amount: Float, currency: Str) -> Str {
    /// Format amount as currency
    /// 
    /// # Arguments
    /// 
    /// * `amount` - Numeric amount
    /// * `currency` - Currency symbol
    /// 
    /// # Returns
    /// 
    /// Formatted currency string
    currency + Obara.format(amount, 2);
}

// Internal helpers
private ese clean_string(s: Str) -> Str {
    Ika.trim(Ika.replace(s, "  ", " "));
}

Awesome utility functions for Ifá-Lang providing common string manipulation, data validation, and formatting utilities.

## Features

- **String Utilities**: Title case formatting, trimming, cleaning
- **Email Validation**: RFC-compliant email validation
- **Currency Formatting**: Locale-aware currency formatting
- **Type Safety**: Full type checking and error handling
- **Cross-Platform**: Works on native, WASM, and embedded targets

## Installation

Add to your `ifa.toml`:

```toml
[dependencies]
ifa-awesome-utils = "1.0.0"
```

## Quick Start

```ifa
import ifa_awesome_utils;

// Format a name
ayanmo name = ifa_awesome_utils::format_name("alice");
Irosu.fo(name); // "Alice"

// Validate email
ayanmo is_valid = ifa_awesome_utils::validate_email("user@example.com");
Irosu.fo(is_valid); // otito

// Format currency
ayanmo price = ifa_awesome_utils::format_currency(123.45, "$");

```

# API Documentation

## Core Functions

### String Functions

#### `format_name(name: Str) -> Str`

Formats a name to title case by capitalizing the first letter and making the rest lowercase.

**Parameters:**
- `name` (Str): Input name string

**Returns:** (Str) Title-cased name

**Throws:** `IfaError::TypeError` if input is not a string

**Examples:**
```ifa
format_name("alice")    // "Alice"
format_name("ALICE")    // "Alice"
format_name("aLiCe")    // "Alice"
```

#### `clean_string(s: Str) -> Str` (Internal)

Removes extra whitespace and normalizes spacing.

**Parameters:**
- `s` (Str): String to clean

**Returns:** (Str) Cleaned string

**Note:** This is an internal function, not part of the public API.

### Validation Functions

#### `validate_email(email: Str) -> Bool`

Validates email address using basic format checking.

**Parameters:**
- `email` (Str): Email address to validate

**Returns:** (Bool) `otito` if valid, `eke` otherwise

**Validation Rules:**
- Contains exactly one `@` symbol
- Contains at least one `.` after `@`
- Not empty string

**Examples:**
```ifa
validate_email("user@example.com")  // otito
validate_email("invalid-email")      // eke
validate_email("")                   // eke
```

### Formatting Functions

#### `format_currency(amount: Float, currency: Str) -> Str`

Formats a numeric amount as currency with two decimal places.

**Parameters:**
- `amount` (Float): Numeric amount
- `currency` (Str): Currency symbol

**Returns:** (Str) Formatted currency string

**Examples:**
```ifa
format_currency(123.45, "$")  // "$123.45"
format_currency(0.0, "€")      // "€0.00"
format_currency(-50.25, "£")    // "£-50.25"
```

## Error Handling

All functions follow Ifá-Lang error handling conventions:

```ifa
gbiyanju {
    ayanmo result = format_name(input);
} gba (error) {
    Irosu.fo("Error formatting name: " + error.message);
}
```

## Performance Considerations

- `format_name`: O(n) complexity, where n is string length
- `validate_email`: O(n) complexity, basic string operations
- `format_currency`: O(1) complexity, uses Obara domain formatting

## Thread Safety

All functions are thread-safe and can be used in concurrent environments.

Semantic Versioning

Follow semantic versioning (SemVer) for package versions:

  • MAJOR (X.0.0): Breaking changes
  • MINOR (X.Y.0): New features, backward compatible
  • PATCH (X.Y.Z): Bug fixes, backward compatible

ifa-docs = "*"  # Any version

# Greater than or equal to
ifa-cli = ">=1.2.0"

ifa-babalawo = ">=1.0.0, <2.0.0"

ifa-web = { version = ">=1.0.0", features = ["async"] }

Troubleshooting

Common Publishing Issues

Name Already Taken

Error: Package name is already registered

Solution: Choose a different name or check if you own the existing package

Invalid Version Format

Error: Version doesn't follow semantic versioning

Solution: Use format MAJOR.MINOR.PATCH (e.g., 1.0.0)

Missing Files

Error: Required files are missing

Solution: Ensure README.md, LICENSE, and ifa.toml are present

Dependency Conflicts

Error: Dependencies have conflicting requirements

Solution: Update dependency versions or resolve conflicts

Authentication Failed

Error: Invalid credentials or authentication error

Solution: Check username/password or re-authenticate with `ija login`
