?? Oja Publishing Guide

Oja (Yoruba: "market") is the Ifá-Lang package manager. This guide shows how to create and publish packages.

Package Structure

my-package/
+-- oja.toml          # Package manifest
+-- src/
¦   +-- lib.ifa       # Main library entry
+-- tests/
¦   +-- test_lib.ifa  # Tests
+-- examples/
¦   +-- demo.ifa      # Examples
+-- README.md         # Documentation

oja.toml Manifest

[package]
name = "my-package"
version = "1.0.0"
description = "A useful Ifá-Lang package"
authors = ["Your Name "]
license = "MIT"
repository = "https://github.com/you/my-package"
keywords = ["utility", "helper"]

[dependencies]
other-package = "2.1.0"
another-lib = { version = "1.0", features = ["async"] }

[dev-dependencies]
test-utils = "0.5.0"

Creating a Package

# Create new package
ifa new my-package
cd my-package

# Or initialize in existing directory
ifa init

Versioning

Follow semantic versioning (SemVer):

  • MAJOR — Breaking changes
  • MINOR — New features, backwards compatible
  • PATCH — Bug fixes
# Bump version
ifa version patch  # 1.0.0 ? 1.0.1
ifa version minor  # 1.0.1 ? 1.1.0
ifa version major  # 1.1.0 ? 2.0.0

Publishing

# Login to registry
ifa login

# Run checks before publishing
ifa check
ifa test

# Publish to registry
ifa publish

Pre-publish Checklist

  • All tests pass
  • No Babalawo warnings
  • README.md is up to date
  • Version number is correct
  • Dependencies are pinned

Installing Packages

# Add dependency
ifa add package-name

# Add with specific version
ifa add package-name@2.0.0

# Install all dependencies
ifa install

Using Packages

// Import the package
lo my-package

// Or import specific items
lo { helper, utility } lati my-package

// Use in code
ayanmo result = helper.do_something();

Private Packages

# Publish private package
ifa publish --private

# Use private registry
[registry]
url = "https://private.registry.example.com"
token = "$OJA_PRIVATE_TOKEN"
