Opon - Memory Configuration

Opon (Sacred Calabash) configures VM memory with runtime enforcement.

Directive Syntax

// At top of file
#opon kekere;    // Small: 256 slots (~4KB)
#opon arinrin;   // Standard: 4096 slots (~64KB)
#opon nla;       // Large: 65536 slots (~1MB)
#opon ailopin;   // Unlimited: grows dynamically

// English aliases work too
#opon small;
#opon large;

Size Options

Yoruba English Slots Memory Use Case
kekere small, tiny, embedded 256 ~4KB IoT, embedded
arinrin medium, standard, default 4,096 ~64KB CLI tools, scripts
nla large, big, mega 65,536 ~1MB Web servers, data
ailopin unlimited, dynamic Dynamic Grows Development
Enforced! If memory limit is exceeded, operations fail with helpful error: 
Opon memory limit exceeded: requested slot 257 but limit is 256 (250 slots in use).
Hint: Use '#opon nla' or '#opon ailopin' for more capacity.

Rust API

use ifa_core::{Opon, OponSize, OponError};

// Create with size
let mut opon = Opon::new(OponSize::Kekere);

// Safe operations
opon.try_set(addr, value)?;  // Returns OponResult
let addr = opon.allocate(value)?;

// Query capacity
opon.can_allocate(10);       // Check before bulk insert
opon.remaining_capacity();   // Slots left

CSS Support (Future Enhancement)

Currently Ose renders ASCII art only. CSS support would require modifications:

Why CSS isn't supported:
