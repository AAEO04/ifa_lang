Opele Chain & Divination

Unique Ifá-Lang features inspired by Yoruba divination

v1.2.1

The Concept

In traditional Ifá divination, the Opele is a divination chain used by Babalawos (priests) to cast readings. The chain produces binary patterns (open/closed) that form the Odù - sacred verses containing wisdom and guidance.

Ifá-Lang brings this concept to computing: random generation with meaning, tamper-evident logging, and pattern-based decision making.

Features

Opele.cast()

Cast a random Odù using cryptographically secure RNG.

ayanmo odu = Opele.cast();
Irosu.fo(odu);
// Output: "Ogunda-Ose"

Implemented in v1.0

Opele.cast_compound(n)

Cast multi-level compound Odù (2-5 levels).

ayanmo gpc = Opele.cast_compound(3);
// 3-level genealogical pattern
Irosu.fo(gpc.lineage());

Implemented in v1.2

Opele.divine(question)

Get guidance based on cast Odù patterns.

ayanmo advice = Opele.divine(
  "Should I proceed?"
);
Irosu.fo(advice);

Implemented in v1.1

Opele.lineage(compound)

Get genealogical description of compound Odù.

ayanmo desc = Opele.lineage(gpc);
// "Ogunda born of Ose, 
//  grandchild of Iwori"

Implemented in v1.2

OpeleChain: Tamper-Evident Logging

Inspired by blockchain technology, OpeleChain provides an append-only, tamper-evident log. Each entry is linked to the previous via cryptographic hashing, making any modification detectable.

How It Works

Genesis
hash: 0x0000
→
Entry 1
prev: 0x0000
→
Entry 2
prev: 0xa1b2
→
Entry 3
prev: 0xc3d4

Usage

// Create a new chain
ayanmo chain = OpeleChain.new();

// Append entries (returns hash)
ayanmo h1 = chain.append("User logged in");
ayanmo h2 = chain.append("Transaction: 100 NGN");

// Verify chain integrity
ti chain.verify() {
    Irosu.fo("Chain is valid");
} bibeko {
    Irosu.fo("TAMPERING DETECTED!");
}

// Get entry by hash
ayanmo entry = chain.get(h1);
Irosu.fo(entry.data);  // "User logged in"

ase;

Iwa Balance: Resource Lifecycle

The Iwa (character/ethics) engine enforces balanced resource handling. Every "open" must have a "close" - this is the principle of Iwa-Pele (good character).

Open Close Domain
Odi.si() Odi.pa() File I/O
Otura.so() Otura.pa() Network
Osa.khoa() Osa.si() Locks
Irete.ya() Irete.tu() Memory

Use ebo (sacrifice/cleanup guard) for automatic cleanup:

ebo file = Odi.si("data.txt", "r") {
    // File automatically closed when scope ends
    ayanmo content = Odi.ka(file);
    Irosu.fo(content);
}
// file.pa() called automatically here
