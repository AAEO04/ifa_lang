🔧 CPU Infrastructure

The Scheduler — Task Parallelism & Async Execution

Overview

CPU Infrastructure provides high-performance task parallelism and async execution capabilities. Built on Rayon thread pool with tokio async runtime integration, it enables efficient multi-threaded computation and concurrent I/O operations.

Core Components

CpuContext

Global CPU context providing unified access to thread pool and async runtime:

CPU Configuration 
// Configure global thread pool
CpuContext.configure(8)?;  // 8 worker threads

// Execute blocking computation
let result = CpuContext.run_blocking(|| {
    // CPU-intensive work
    expensive_calculation()
});

// Execute async operation
let async_result = CpuContext.run_async(|| {
    async {
        fetch_data().await
    }
});

Features

🚀 Task Parallelism

  • Rayon Integration: High-performance data parallelism
  • Thread Pool: Configurable worker threads
  • Work Stealing: Automatic load balancing
  • Fork-Join: Parallel divide-and-conquer algorithms

⚡ Async Execution

  • Tokio Runtime: Production-grade async executor
  • Multi-threaded: 2 dedicated async worker threads
  • Blocking Bridge: Seamlessly mix sync/async code
  • I/O Integration: Efficient concurrent I/O operations

🔒 Thread Safety

  • Send + Sync: All data types are thread-safe
  • Arc/Mutex: Shared state protection
  • Atomic Operations: Lock-free counters and flags
  • Memory Ordering: Proper synchronization semantics

API Reference

CpuContext::configure(threads: usize)

Configure the global thread pool with specified number of worker threads.

  • threads: Number of worker threads (recommended: CPU cores)
  • Returns: Result<(), String> - error if thread pool creation fails

CpuContext::run_blocking<F, R>(func: F) -> R

Execute a blocking function in the thread pool and wait for completion.

  • func: Closure to execute (must be Send)
  • Returns: R - function result
  • Use Case: CPU-intensive computations

CpuContext::run_async<F, Fut, R>(f: F) -> R

Execute an async future using the shared tokio runtime (blocking bridge).

  • f: Closure returning async future
  • Returns: R - future result
  • Use Case: Mixing sync and async code

Performance Characteristics

Thread Pool Management

  • Global Instance: Single thread pool for all operations
  • Lazy Initialization: Created on first use
  • Work Stealing: Automatic load balancing across threads
  • Graceful Shutdown: Clean thread termination

Memory Efficiency

  • Zero Allocation: Thread pool reuse
  • Stack-based: Minimal heap allocations
  • CPU Cache: Thread-local data structures
  • NUMA Aware: Thread affinity optimization

Usage Examples

Parallel Data Processing 
// Process large dataset in parallel
ese process_data_parallel(data: Vec) -> Vec {
    CpuContext.run_blocking(|| {
        data.par_iter()
            .map(|x| x * x + 1)
            .filter(|&x| x % 2 == 0)
            .collect()
    })
}

// Mixed sync/async operations
ese mixed_operations() {
    let cpu_result = CpuContext.run_blocking(|| {
        // CPU-bound calculation
        expensive_computation()
    });
    
    let io_result = CpuContext.run_async(|| async {
        // I/O-bound operation
        fetch_remote_data().await
    });
    
    (cpu_result, io_result)
}

Feature Flags

CPU infrastructure is controlled by the parallel feature flag:

Cargo.toml 
[dependencies]
ifa-std = { version = "1.3.0", features = ["parallel"] }

# Optional tokio integration
ifa-std = { version = "1.3.0", features = ["parallel", "tokio"] }
