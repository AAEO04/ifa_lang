💾 Storage Infrastructure

The Seal — High-Performance I/O & Native Persistence

Overview

Storage Infrastructure provides high-performance I/O and native persistence with production-grade durability. Features async file operations, structured key-value storage, automatic checksums, and corruption recovery.

Core Components

StorageError

Comprehensive error handling for all storage operations with detailed error types:

Error Types 
// Storage operation errors
pub enum StorageError {
    Io(std::io::Error),                    // I/O failures
    Serialization(bincode::Error),           // Data serialization
    KeyNotFound,                          // Missing key
    KeyTooLarge(u64),                      // Key size limit exceeded
    ValueTooLarge(u64),                    // Value size limit exceeded
    CorruptedLog(String),                   // Log corruption
    ChecksumMismatch { expected: u32, actual: u32 }, // Data corruption
    InvalidFormat,                         // File format error
}

Constants & Limits

  • MAX_KEY_SIZE: 1 MB (1,048,576 bytes)
  • MAX_VALUE_SIZE: 100 MB (104,857,600 bytes)
  • MAGIC: File format identifier "IFAS"
  • VERSION: Current file format version 2

API Reference

File Operations

Async File I/O 
// Async file operations
use tokio::fs::{File, OpenOptions};

// Open file with options
let mut file = OpenOptions::new()
    .read(true)
    .write(true)
    .create(true)
    .open("data.bin").await?;

// Async read
let mut buffer = vec![0u8; 1024];
let bytes_read = file.read(&mut buffer).await?;

// Async write with seek
file.seek(SeekFrom::Start(0)).await?;
file.write_all(&data).await?;

File Format

IFAS Format Structure

  • Header: Magic bytes (4) + Version (1)
  • Entries: Key length + Value length + Key + Value + Checksum
  • Index: Optional key-to-offset mapping
  • Footer: Entry count + format checksum
File Format Example 
// IFAS file format (little-endian)
struct IfasHeader {
    magic: [u8; 4],      // "IFAS"
    version: u8,            // Format version (2)
    reserved: [u8; 3],    // Future use
}

struct IfasEntry {
    key_len: u32,          // Key length
    value_len: u64,        // Value length  
    key: Vec,         // Key data
    value: Vec,       // Value data
    checksum: u32,        // CRC32 of key + value
}

Error Handling

Recovery Strategies

  • Checksum Mismatch: Attempt data recovery
  • Corrupted Log: Truncate to last valid entry
  • Invalid Format: Create new storage file
  • I/O Errors: Retry with exponential backoff

Debugging Support

  • Detailed Errors: Contextual error messages
  • Error Codes: Machine-readable error types
  • Logging Integration: Structured error logging
  • Recovery Logs: Automatic corruption reporting

Usage Examples

Complete Storage Example 
// High-performance data persistence
ese user_data_manager() -> Result<(), StorageError> {
    let store = Storage::new("users.ifas").await?;
    
    // Insert user with automatic serialization
    let user = User {
        id: 123,
        name: "Alice".to_string(),
        email: "alice@example.com".to_string(),
    };
    
    store.put(&format!("user:{}", user.id), &user).await?;
    
    // Query users with pattern
    let users: Vec = store
        .scan_prefix("user:")
        .await?
        .into_iter()
        .filter_map(|(_, user)| user.ok())
        .collect();
    
    Irosu.fo($"Found {users.len()} users");
    
    // Cleanup old data
    store.cleanup_old_entries(Duration::from_days(30)).await?;
    
    Ok(())
}
