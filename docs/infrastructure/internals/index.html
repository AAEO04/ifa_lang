This section documents the internal architecture of Ifá-Lang for contributors and advanced users, based on the actual codebase implementation and runtime specification.

Three-Tier Runtime Architecture

Ifá-Lang implements a unified three-tier execution model as specified in the runtime specification:

AST Interpreter

Direct source code interpretation for development and debugging

Source: ifa-core/src/parser.rs, interpreter/core.rs

Bytecode VM

Optimized stack-based bytecode execution for production

Source: ifa-core/src/vm.rs, compiler.rs

Transpiler

Native code generation for maximum performance

Source: ifa-core/src/transpiler/

Architecture Principles

Protocol-Oriented Design

  • No Classes: Trait-based composition instead of inheritance
  • Structural Typing: Capability-based type checking
  • Domain Protocols: Odù-specific capability contracts
  • Composition: Flexible trait composition patterns

Cultural Integration

  • Sacred Memory: Opon system with divination tray metaphor
  • Resource Obligation:  (sacrifice) lifecycle management
  • Community Relationships: Àjõsé reactive system
  • Gentle Balance: Ìwà Pélé harmony enforcement

Performance Philosophy

  • Zero Cost Abstractions: Compile-time optimizations
  • Runtime Flexibility: Dynamic typing with performance
  • Memory Efficiency: Sacred space management
  • Reactive Efficiency: Minimal overhead relationships

Contributing to Internals

Development Guidelines

  • Cultural Alignment: Maintain Yoruba philosophical principles
  • Protocol Consistency: Follow protocol-oriented design patterns
  • Performance Awareness: Consider zero-cost abstractions
  • Documentation: Update this documentation with changes

Testing Strategy

  • Unit Tests: Individual component testing
  • Integration Tests: Cross-component interaction
  • Performance Tests: Benchmark critical paths
  • Cultural Tests: Verify philosophical alignment

Code Organization

  • Single Responsibility: Each module has clear purpose
  • Dependency Direction: Core types at bottom, features on top
  • Feature Flags: Conditional compilation for capabilities
  • Error Handling: Canonical IfaError usage throughout
