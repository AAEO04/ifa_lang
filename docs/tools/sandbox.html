The ifa-sandbox crate provides a deny-by-default security model for running untrusted Ifá-Lang code safely.

?`fún Capability System

No code can access system resources without explicit capability grants:

enum Ofun

enum Ofun {
    // File system
    FileRead,
    FileWrite,
    FileDelete,
    
    // Network
    NetConnect,
    NetListen,
    NetSend,
    
    // System
    EnvRead,
    EnvWrite,
    ProcessSpawn,
    
    // Hardware (for embedded)
    GpioAccess,
    SerialAccess,
}

CapabilitySet

CapabilitySet

Collection of granted capabilities.

let mut caps = CapabilitySet::new();
caps.grant(Ofun::FileRead);
caps.grant(Ofun::NetConnect);

// Check permission
if caps.has(Ofun::FileWrite) {
    // allowed
}

Security Profiles

Profile Capabilities Use Case
Minimal None Pure computation
ReadOnly FileRead, EnvRead Config reading
Network NetConnect, NetSend API clients
Development All except ProcessSpawn Local dev
Unrestricted All Trusted code only 
let config = SandboxConfig::new()
    .profile(SecurityProfile::ReadOnly)
    .memory_limit(64 * 1024 * 1024)  // 64MB
    .cpu_limit_ms(5000);             // 5 second timeout

OmniBox (WASM Isolation)

Complete isolation via WebAssembly:

let omnibox = OmniBox::new(config);
let result = omnibox.run(bytecode)?;

// Code runs in WASM sandbox
// - No direct memory access
// - All I/O goes through capability-checked host functions

ResourceMonitor

Track and limit resource usage:

let monitor = ResourceMonitor::new();

// Get current usage
let stats = monitor.stats();
println!("Memory: {} bytes", stats.memory_used);
println!("CPU time: {} ms", stats.cpu_time_ms);
println!("Files opened: {}", stats.files_opened);
println!("Network bytes: {}", stats.network_bytes);

CLI Usage

# Run with minimal sandbox
ifa run --sandbox=minimal script.ifa

# Run with specific capabilities
ifa run --sandbox --allow=file:read,net:connect script.ifa

# Run in WASM isolation
ifa run --omnibox script.ifa
