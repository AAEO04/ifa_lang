Set up your editor for Ifá-Lang development

Language Server Protocol (LSP)

Ifá-Lang includes a built-in LSP server for IDE integration.

Starting the LSP Server

ifa lsp

Features

  • Syntax highlighting — Recognized keywords and Odù domains
  • Auto-completion — Domain methods, keywords, variables
  • Diagnostics — Parse errors, type warnings from Babalawo
  • Document sync — Real-time error checking

Visual Studio Code

Extension Installation

# Install from VS Code Marketplace (when available)
code --install-extension ifa-lang.ifa-lang

# Or manually from VSIX
code --install-extension ifa-lang-0.1.0.vsix

settings.json

{
    "ifa.linter.enabled": true,
    "ifa.linter.strict": false,
    "ifa.format.onSave": true,
    "ifa.lsp.path": "ifa",
    "ifa.lsp.args": ["lsp"],
    "[ifa]": {
        "editor.tabSize": 4,
        "editor.insertSpaces": true,
        "editor.formatOnSave": true
    }
}

Manual LSP Configuration

If not using the extension, configure with a generic LSP client:

// settings.json
{
    "lsp.servers": {
        "ifa": {
            "command": "ifa",
            "args": ["lsp"],
            "filetypes": ["ifa"]
        }
    }
}

Neovim

With nvim-lspconfig

-- init.lua or lspconfig setup
local lspconfig = require('lspconfig')
local configs = require('lspconfig.configs')

-- Define Ifá-Lang LSP
if not configs.ifa_babalawo then
    configs.ifa_babalawo = {
        default_config = {
            cmd = { 'ifa', 'lsp' },
            filetypes = { 'ifa' },
            root_dir = lspconfig.util.root_pattern('oja.toml', '.git'),
            settings = {},
        },
    }
end

lspconfig.ifa_babalawo.setup{
    on_attach = function(client, bufnr)
        -- Enable completion
        vim.api.nvim_buf_set_option(bufnr, 'omnifunc', 'v:lua.vim.lsp.omnifunc')
    end,
}

Filetype Detection

-- ftdetect/ifa.lua
vim.filetype.add({
    extension = {
        ifa = 'ifa',
    },
})

Syntax Highlighting (TreeSitter)

-- Add to your TreeSitter config when parser is available
require('nvim-treesitter.configs').setup {
    ensure_installed = { 'ifa' },
    highlight = { enable = true },
}

Emacs

With lsp-mode

;; Add to init.el
(use-package lsp-mode
  :hook (ifa-mode . lsp)
  :commands lsp)

(lsp-register-client
  (make-lsp-client
    :new-connection (lsp-stdio-connection '("ifa" "lsp"))
    :major-modes '(ifa-mode)
    :server-id 'ifa-babalawo))

;; Basic ifa-mode
(define-derived-mode ifa-mode prog-mode "Ifá"
  "Major mode for editing Ifá-Lang files."
  (setq-local comment-start "// ")
  (setq-local comment-end ""))

(add-to-list 'auto-mode-alist '("\\.ifa\\'" . ifa-mode))

JetBrains IDEs

IntelliJ / WebStorm / PyCharm

  1. Install "Generic LSP Client" plugin
  2. Go to Settings ? Languages & Frameworks ? LSP
  3. Add new server:
    • Name: Ifá-Lang
    • Command: ifa lsp
    • File patterns: *.ifa

Sublime Text

With LSP Package

// Preferences ? Package Settings ? LSP ? Settings
{
    "clients": {
        "ifa": {
            "enabled": true,
            "command": ["ifa", "lsp"],
            "selector": "source.ifa"
        }
    }
}

File Association

Ifá-Lang source files use the .ifa extension:

*.ifa          ? Ifá-Lang source code
*.ifab         ? Ifá-Lang bytecode (compiled)
oja.toml       ? Project manifest
